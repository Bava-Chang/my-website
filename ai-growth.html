用 AI 建立我的學習地圖：從 0 到 1 的關鍵心法

學習策略AI心法

我把學習拆成三層：工具（會用）、方法（會想）、系統（能複製）。一開始只選一個小專案，例如「把每日產線數據自動匯總」；完成後寫下 SOP 與檢核表，讓未來的我可以更快複製成功。

展開重點
  • 設定可衡量的成果：輸入→處理→輸出
  • 優先做 對自己有用 的題目，動機最強
  • 把「踩坑」記錄成 FAQ，未來少踩一次

工廠數位轉型藍圖：從無紙化到智慧排程

智慧工廠MES排程

藍圖分三步：數據化（表單化/掃碼）、可視化（看板/告警）、智慧化（建議/自動化）。以出貨、烘房、換模、品檢為四大主線，逐一建立資料標準與 API 介面。

輸入統一，報表自然穩定，決策才能快。

Google Apps Script：把表單、Sheets 與現場流程串起來

Apps ScriptWeb App表單

用 doGet/doPost 建立 Web App 入口；前端用 HTML/CSS/JS，後端寫入 Sheets。把「點檢表設定」→「動態生成表單」→「寫入紀錄」→「分析與看板」串成一條龍，讓現場只要掃碼就能上傳。

我學到的重點
  • 欄位命名先定義：英數、無空白、與報表一致
  • 錯誤一定記 debug_log，才找得到原因
  • Web App 部署權限需設定「任何知道連結的人」

Node‑RED 入門：把資料接起來，做出可視化儀表板

Node‑REDDashboardIoT

把掃碼、空壓機、烘房與庫存，透過 Node‑RED 串接：Inject → Function → Switch → Dashboard。我先做一個「入烘紀錄」小流程，再逐步擴充到告警與周報。

可視化不是漂亮，是讓人「一眼看懂」。

Python 打造自動化報表：Pandas 核心技巧

PythonPandas報表

把原本 VBA 的月報邏輯，用 Pandas 重寫：read_excel → groupby → pivot_table → 輸出 ExcelWriter。形成可重複、可版本化的腳本，降低人工出錯。

固定模板的威力
  • 所有欄位英文代碼化，避免空白與全半形問題
  • 月報自動產生多個工作表，名稱帶日期
  • 超過 80% 分位的極端值另外標示，圖表不失真

ISO 與品質管理：用數據說話的 8D 與 PDCA

ISO8DPDCA

客訴處理我固定用 8D：客訴事實 → 臨時對策 → 根因 → 永久對策 → 成效驗證。所有附件（照片、測試、簽到）一律數位化，關鍵欄位對應到「品質履歷」表。

沒有資料，就只有說服；有了資料，自然有答案。

空壓機與能源管理：從流量、稼動率到節能 ROI

能源管理空壓機成本

建立「重車/空車」與用氣量的資料表，分別計算日/週/月電費；提出「啟停策略」與「旁路分流」方案，並用 Excel/Sheets 做情境比較，讓投資回收期（ROI）一目了然。

掃碼入烘系統：QR × Web App × Google Sheets 全流程

QRWebSheets

流程是：掃「烘房 QR」→ 開相機 → 掃「台車 QR」→ 自動帶入表單 → 一鍵送出 → 寫入 dryroom_log。畫面完成送出後自動返回相機，便於連續操作。

員工績效看板：任務派工、權重與達成率

績效看板任務派工

任務拆解為「工單」與「子任務」，每項有「權重、開始/截止、完成勾選、備註」。主管清空任務只影響主管視圖，廠長仍可追蹤；所有動作都有歷史紀錄。

習慣養成系統：健身、冥想、飲食與睡眠的資料化

習慣自我管理健康

我設定每日固定任務：早上 30 分鐘運動、晚上 30 分鐘運動、不吃檳榔、冥想 10 分鐘、23 點前睡、自煮中晚餐。用簡單的打卡表與趨勢圖，讓紀律變成可視化的反饋。
