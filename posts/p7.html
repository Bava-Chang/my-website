Node-RED：把資料「流」起來的第一張看板

源由｜目的｜過程｜困難與修正｜結論與建議

我用 inject → function → dashboard 的最小流程，先把單一變數跑起來，接著再接設備訊號。AI 協助釐清每個節點用途並產生初始 dashboard 配置。

困難與修正

  • 套件版本：選官方 dashboard 套件，避免社群分歧。
  • 時間戳顯示：function node 格式化後再丟 template。
  • 部署灰掉：檢查 flow 是否有未連線節點。

結論

先跑通「一條乾淨的資料流」，再擴展多訊號，學習曲線會平順很多。
