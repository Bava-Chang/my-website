AI × 品質異常分析：8D 的自動生成

當客戶要求 8D Report 不再需要加班熬夜時，AI 不只是幫我寫報告，而是讓品保從「填表人」變成「問題設計師」。

一、源由：客訴的壓力與8D的煩惱

每當客戶發來「變形」「破損」「含水」等客訴時， 品保部門必須在 24 小時內回報 8D。 一份 8D 需要八個步驟：團隊組成、問題描述、圍堵措施、根本原因、永久改善、預防再發、驗證效果、關閉報告。 這在 ISO 系統裡是基本功，但在現場卻是耗時工作。 我想讓這件事變成自動化：AI 幫我生成初稿、人員只需修訂與上傳證據。

8D 痛點
耗時 每次重寫內容雷同；
缺證據 照片、教育訓練、簽名漏傳；
不一致 不同工程師格式不同；
追蹤難 8D 改善後無法連動教育訓練紀錄。

二、目的：讓AI理解問題結構，而非生成文字

我並不希望 AI 幫我「編故事」， 而是希望它理解品質事件的結構化資料： 何時、哪個產品、哪個機台、異常現象、暫時處置、最終改善。 只要把這些欄位輸入，AI 就能生成具 ISO 格式的 8D 初稿， 並自動帶出相關教育訓練與追蹤項目。

三、過程：從異常輸入到8D輸出

核心概念：一筆異常資料，對應一份8D + 一份教育訓練 + 一份改善追蹤。

3.1 結構化輸入表（Google Sheet）

日期 | 客戶 | 料號 | 異常描述 | 原因 | 暫時措施 | 永久改善 | 負責人 | 照片連結
2025/07/29 | 友達 | 3MDL9NPK020 | 變形、破損 | 模溫過高 | 全檢＋降溫 | 模具更新＋教育訓練 | 品保課 | GDrive連結

3.2 AI 生成 8D Report（Prompt 範例）

你是品質系統工程師，請根據下列資料生成 8D 報告：
1) 結構：D1~D8，符合 ISO9001 要求。
2) 用詞正式、簡潔、可供客戶審查。
3) 自動帶出對應教育訓練內容與追蹤措施。
4) 以 HTML 輸出，段落分明，標題粗體。
5) 輸入資料如下：
客戶：友達
異常：變形4pcs、破損1pcs
原因：模溫過高
暫時措施：降溫、全檢
永久改善：模具更新、教育訓練

3.3 Apps Script 範例：自動產出與歸檔

function gen8D() {
  const sh = SpreadsheetApp.getActive().getSheetByName('異常登錄');
  const rows = sh.getDataRange().getValues().slice(1);
  for (let r of rows) {
    const [date, customer, part, desc, cause, action, fix, owner] = r;
    const payload = {date, customer, part, desc, cause, action, fix, owner};
    const aiText = callOpenAI(payload);  // 傳送給 ChatGPT API
    const doc = DocumentApp.create(`${date}_${customer}_${part}_8D`);
    doc.getBody().appendParagraph(aiText);
    doc.saveAndClose();
  }
}

四、挑戰與修正

  • AI 過度簡化：生成內容太「教科書」。→ 增加實際資料欄位（溫度、壓力、時間）。
  • 責任歸屬模糊：AI 不懂部門分工。→ 加入責任人欄與流程控制。
  • 報告不一致：不同客戶格式不同。→ 建立 AI 模板（客戶別規格）。
  • 缺乏追蹤：8D 完成後未回饋訓練。→ 建立「關聯課程」欄位自動觸發教育訓練。
上線後成果
8D 撰寫時間由 2 小時 → 10 分鐘； 品保文件一致性提升至 95%； 教育訓練與矯正追蹤自動連動； 客戶審核通過率提高，回報速度加快一倍。

五、我用的第二階段 Prompt（AI 校稿模式）

你是 ISO 品質系統審查員，請幫我審查以下 8D：
1) 語氣是否正式、專業
2) 是否符合 ISO9001 條文 10.2 不合格處理
3) 是否有 D4 根因與 D5 永久措施連貫性
4) 若有缺漏，請以「修訂建議」方式列出

六、結論：AI 不只是生成，而是「系統化經驗」

品質管理的核心從來不是文件，而是思考。 8D 的價值不在於報告，而在於團隊「如何找出根因」。 AI 幫我們節省了格式與書寫時間， 讓人可以專注於「為什麼會發生」與「怎麼防止再發」。 真正的智慧品保，不是會自動寫報告的系統， 而是能讓整個公司都懂得如何回答「為什麼」的文化。
