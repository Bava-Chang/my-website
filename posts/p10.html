AI × 工安事件預防：從風險辨識到預測模型

事故從不會突然發生，它早就在數據裡留下痕跡。這篇記錄我如何用 AI 找出風險模式，預測工安事件，讓安全從「反應式」轉為「預測式」。

一、源由：工安事故背後的「隱性變數」

每次工安事故發生，報告都寫著「員工未遵守 SOP」、「教育訓練不足」。 但如果仔細回顧數據，你會發現：事故前幾天總有共同特徵。 WBGT 值偏高、噪音超標、工時過長、設備異常未修復、甚至連加班紀錄都能預示危險。 我開始思考：能不能讓系統自己看出這些模式？

資料來源整合
環境數據 WBGT、溫度、濕度、噪音；
人因數據 工時、班表、加班時數；
設備數據 停機、故障、壓力、溫控異常；
教育訓練 最近訓練項目與完成率。

二、目的：從紀錄轉為預測

傳統的工安系統只會在「事件發生後」反應， 而我希望它能在「事件發生前」提醒。 我設定三個目標：

  • 自動收集各類安全相關數據並標記事件前後變化。
  • 用機器學習建立「高風險班次預測模型」。
  • 將結果回饋到看板與教育訓練排程中。

三、過程：AI 從資料中學會「危險感」

我建立了資料管線，讓系統每天自動更新，並以 Python 訓練模型。

3.2 Python 預測模型（邏輯迴歸範例）

import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression

df = pd.read_csv('safety.csv')
X = df[['WBGT','噪音','工時','停機數']]
y = df['事故'].map({'是':1,'否':0})
X_train, X_test, y_train, y_test = train_test_split(X,y,test_size=0.3,random_state=42)

model = LogisticRegression()
model.fit(X_train, y_train)

df['風險指數'] = model.predict_proba(X)[:,1]
df.to_csv('result.csv', index=False)
print("模型訓練完成，風險分數已更新。")

我在每日更新流程中加入這段程式， 系統會依據最新資料自動評估每個班次的風險分數。 若超過門檻（例如 0.7），就會觸發警報與教育訓練提醒。

四、挑戰與修正

  • 資料不一致：不同表單單位不同。→ 統一單位（°C、dB、hr）。
  • 標籤稀少：事故樣本太少。→ 改用「異常」欄位作為間接標籤。
  • 誤報過多：高溫但未出事也被警告。→ 加入「教育完成率」作為風險修正項。
  • 無法即時預測：資料更新慢。→ 使用 Apps Script + Python API 每日同步。
上線後成效
模型準確率達 87%； 高風險日提前通報 2~3 天； 事故件數下降 46%； 教育訓練針對性提升，現場回饋更明確。

五、我用的 Prompt（模型輔助）

你是工安數據分析師，請根據以下資料建立預測模型：
1) 資料欄位：WBGT、噪音、工時、停機數、事故
2) 請建立 Logistic Regression 模型
3) 輸出每筆資料的「風險指數」(0~1)
4) 若風險指數 > 0.7，給出建議行動：
   - 教育訓練
   - 通風改善
   - 工時調整
   - 班表變更
5) 以 CSV 輸出結果

六、結論：安全管理的未來，是「可預測的預防」

AI 不會取代安全主管，但它能讓風險「被看見」。 從資料收集、模型訓練到教育回饋， 工安不再只是檢查表，而是一套會學習的系統。 未來我希望把這模型與 Node-RED 看板結合， 讓風險分數即時顯示在工廠地圖上， 當氣候、溫度、工時超標時自動亮燈、發通知， 讓每一次警示都提前一步。 這不是 AI 在管安全，而是我們用 AI 讓安全變得更聰明。
