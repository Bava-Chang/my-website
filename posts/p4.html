製令成本估算：把 Excel 規則抽象化，再交給 Python 批次

源由｜目的｜過程｜困難與修正｜結論與建議

源由

Excel 能算，但面對多製令、多版本參數時就會崩。我要的是「一鍵批次」、可追蹤、可回溯。

目的

  • 把成本邏輯（電費、工時、報廢率、耗材）獨立成 YAML/JSON 參數。
  • 用 Python 讀取製令清單，逐筆計算，產出彙總與敏感度分析。
  • 保留「假設場景」：例如電價上調 10%、空車時間縮短 20%。

過程與 Prompt

先在 Excel 將公式「命名化」，對應到 Python 函式，再讓 AI 產生批次處理骨架。

關鍵 Prompt（精簡版）
請用 Python 寫一個成本批次工具：
- 讀取 ./orders.csv 與 ./params.yaml
- 對每筆製令計算：原料 + 能耗 + 人工 + 報廢
- 另產生兩份附檔：sensitivity.csv（針對電價、空車率）、summary.csv（各製令總表）

困難與修正

  • 單位不一致：在載入時做單位標準化（kWh、kg、hr）。
  • 邏輯漂移：每次改動參數都要留版本；Python 以檔名帶日期。
  • 驗證：抽樣與 Excel 人工計算比對，偏差超過閾值就標示。

結論與建議

抽象化 + 批次化 = 把成本算清楚。下一步可加入工單實際回饋，變成「預算 vs 實績」。
